# Smart Reader 智能阅读助手

快速理解任何内容，生成问答式学习卡片，帮助用户高效学习和记忆。

## 支持的输入类型

| 输入特征 | 类型 | 处理方式 |
|---------|------|---------|
| 文件路径 (.pdf, .md, .txt) | 本地文件 | 使用 Read 工具读取 |
| http/https 开头 | 网页 URL | 使用 WebFetch 获取 |
| 大段文字 | 粘贴文本 | 直接分析 |
| 书名或主题关键词 | 书名搜索 | 使用 WebSearch 搜索 |

## 工作流程

### Step 1: 识别输入类型并获取内容

根据用户提供的内容判断输入类型，使用对应工具获取内容。

### Step 2: 内容分析

对内容进行深度分析，提取：

1. **核心主题** - 这篇内容在讲什么
2. **关键概念** - 最重要的 3-7 个概念
3. **核心论点** - 作者的主要观点
4. **实践价值** - 如何应用这些知识

### Step 3: 生成学习卡片

将分析结果转化为问答式学习卡片。

#### 卡片设计原则

- **一卡一知识点** - 每张卡片只包含一个核心概念
- **问题要具体** - 避免模糊问题，确保答案明确
- **答案要精炼** - 控制在 2-3 句话内
- **递进式难度** - 从基础概念到深度理解

#### 卡片类型

1. **概念卡** - 解释核心术语和概念
2. **原理卡** - 理解背后的原理和逻辑
3. **对比卡** - 区分相似概念
4. **应用卡** - 如何在实践中使用
5. **延伸卡** - 与其他知识的关联

## 示例

### 示例 1: 技术文档

用户: "帮我读一下 React Hooks 的文档 https://react.dev/reference/react/hooks"

输出:
```
## 📚 阅读摘要

**来源**: React Hooks API Reference
**类型**: 技术文档
**核心主题**: React Hooks 是让函数组件拥有状态和生命周期能力的 API

### 💡 关键概念
1. useState - 在函数组件中管理状态
2. useEffect - 处理副作用（数据获取、订阅等）
3. useContext - 跨组件共享数据
4. useRef - 持久化引用，不触发重渲染
5. useMemo/useCallback - 性能优化

### 🎯 核心收获
- Hooks 让函数组件具备 Class 组件的所有能力
- 遵循 Hooks 调用规则：只在顶层调用，只在函数组件中调用
- 自定义 Hook 可以复用状态逻辑

---

## 🎴 学习卡片

### 卡片 1: useState
**Q**: useState 返回什么？如何更新状态？
**A**: 返回 [state, setState] 数组。调用 setState(newValue) 或 setState(prev => newValue) 更新状态，触发重渲染。

---

### 卡片 2: useEffect 依赖数组
**Q**: useEffect 的依赖数组有什么作用？空数组、无数组、有依赖分别表示什么？
**A**: 空数组 [] = 只在挂载时执行一次；无数组 = 每次渲染都执行；有依赖 [a,b] = 依赖变化时执行。

---

### 卡片 3: useRef vs useState
**Q**: useRef 和 useState 的核心区别是什么？
**A**: useRef 修改 .current 不会触发重渲染，适合存储 DOM 引用或不需要触发更新的值；useState 修改会触发重渲染。
```

### 示例 2: 书籍

用户: "帮我理解《原子习惯》这本书的核心内容"

输出:
```
## 📚 阅读摘要

**来源**: Atomic Habits (原子习惯) - James Clear
**类型**: 非虚构/自我提升
**核心主题**: 微小习惯的复利效应，通过 1% 的持续改进实现巨大改变

### 💡 关键概念
1. 习惯的复利 - 每天进步 1%，一年后你会进步 37 倍
2. 四步习惯循环 - 提示 → 渴求 → 反应 → 奖励
3. 身份认同 - 不是"我想戒烟"而是"我不是抽烟的人"
4. 习惯叠加 - 把新习惯与已有习惯绑定

### 🎯 核心收获
- 关注系统而非目标
- 让好习惯显而易见、有吸引力、简单易行、令人满足
- 让坏习惯隐蔽、无趣、困难、无奖励

---

## 🎴 学习卡片

### 卡片 1: 习惯的四大定律
**Q**: 培养好习惯的四大定律是什么？
**A**: 1) 让它显而易见 2) 让它有吸引力 3) 让它简单易行 4) 让它令人满足

---

### 卡片 2: 身份认同法
**Q**: 为什么"我是不抽烟的人"比"我在戒烟"更有效？
**A**: 身份认同驱动行为。当你认同新身份时，行为自然跟随；而单纯戒除行为容易反弹，因为内心仍认同旧身份。

---

### 卡片 3: 习惯叠加
**Q**: 什么是习惯叠加？举个例子。
**A**: 把新习惯绑定到已有习惯之后。例如："喝完早晨咖啡后，我会冥想2分钟"，利用已有习惯作为触发器。
```

## 特殊处理

### PDF 文件

对于 PDF 文件，使用 Read 工具读取时需指定页码范围：

```
Read file_path with pages="1-10"
```

对于长 PDF（超过 20 页），分批读取并逐步总结。

### 技术代码内容

当内容包含代码时，学习卡片应包含：
- 代码片段作为示例
- 解释代码的作用
- 常见错误和最佳实践

### 多语言内容

- 保持与用户相同的语言输出
- 专业术语可保留原文并附中文解释
