二、准备工作

2.1 你需要什么

  • 一台Mac或Linux电脑（Windows用户需要WSL，稍微麻烦一点）
  • 飞书企业账号（个人免费版也可以，功能够用）
  • 10分钟耐心（真的，只要10分钟）

不需要懂编程，不需要配环境。复制粘贴命令就行。

2.2 Clawdbot是什么？

一句话：Clawdbot是AI助手的"万能遥控器"。

你家可能有好几台电视（比如客厅一台、卧室一台），每台都有自己的遥控器。Clawdbot就像那个万能遥控器，让你用一个工具控制多个AI模型（Qwen、GLM、Minimax等）。

更重要的是，它能把这些AI模型接入飞书、Telegram、Discord等聊天工具。你不用再打开浏览器、登录网页，直接在聊天窗口@机器人就行。

✨ 一个遥控器控制所有电视，一个入口调用所有AI。

推荐用Qwen Code——免费、速度快、中文支持好。对于大部分日常使用，完全够了。

三、第一步：安装Clawdbot（5分钟搞定）

3.1 安装命令

打开终端（Mac用户按Command+Space，输入"Terminal"），复制粘贴下面这行命令：

curl -fsSL https://clawd.bot/install.sh | bash

回车，然后等几秒。

3.3 常见问题

Q1：提示"Permission denied"权限不足？

Q2：网络连接失败？

四、第二步：在飞书创建应用（核心步骤，最复杂但最重要）

这是整个教程最复杂的部分，但别怕。我会把每个按钮在哪里都告诉你。

4.1 进入飞书开放平台

  1. 浏览器打开：https://open.feishu.cn/
  2. 用你的飞书账号登录
  3. 点击右上角的"创建企业自建应用"

【配图3：飞书开放平台首页，标注"创建企业自建应用"按钮位置】

4.2 填写应用基本信息

创建应用后，填写这些信息：

  • 应用名称：比如"Clawdbot助手"（随便起，自己看得懂就行）
  • 应用描述：写一句话，比如"AI智能助手，支持技术问答"
  • 应用图标（可选）：找个机器人图标，或者用默认的

填完点击"确定创建"。

4.3 添加机器人能力

进入应用管理界面，你会看到左侧菜单栏。

  1. 找到"机器人"选项（一般在"应用功能"下面）
  2. 点击"添加机器人"
  3. 填写机器人名称（这个名称会在飞书里显示，比如"Clawdbot"）
  4. 保存

【配图4：标注左侧"机器人"菜单位置和"添加机器人"按钮】

4.4 配置权限（最容易出错的地方）

这一步是核心，也是最容易搞错的。

飞书的权限系统很严格，缺一个权限机器人就不工作。你需要开通3个必需权限：

权限1：im:message.group_at_msg

  • 作用：让机器人能收到群里@它的消息
  • 在哪里：左侧菜单"权限管理" → "消息与群组" → 找到"获取群组中所有消息"
  • 操作：点击"申请"按钮

权限2：im:message.p2p_msg

  • 作用：让机器人能收到私聊消息
  • 在哪里：同样在"消息与群组"里，找到"获取用户发给机器人的单聊消息"
  • 操作：点击"申请"按钮

权限3：im:message:send_as_bot

  • 作用：让机器人能发送回复
  • 在哪里：继续在"消息与群组"里，找到"以应用的身份发消息"
  • 操作：点击"申请"按钮

重要提示：

  • 有些权限会立即开通，有些需要管理员审批
  • 确保这3个权限的状态都是"已开通"，否则机器人不会回复

【配图5：三个必需权限的可视化清单】

✨ 权限配置是最繁琐的，但也是一次配置终身受益的事情。

4.5 配置事件订阅

权限搞定后，让机器人"听到"用户的消息。

  1. 左侧菜单找到"事件订阅"
  2. 你会看到一个"请求地址配置"（也叫回调地址、Callback URL）
  3. 先别填！这个地址要等Clawdbot配置完才有，我们第三步会回来填
  4. 往下滚动，找到"添加事件"
  5. 搜索并勾选："接收消息 v2.0"（事件名：im.message.receive_v1）
  6. 保存

【配图6：标注"事件订阅"菜单位置、"请求地址配置"输入框（标记"暂时留空"）】

4.6 获取凭证（重要！这是Clawdbot连接飞书的钥匙）

现在要拿到两把"钥匙"：

  1. 左侧菜单找到"凭证与基础信息"（有的版本叫"应用凭证"）
  2. 你会看到：
    • App ID（应用ID）：一串字符，比如 cli_a1b2c3d4e5f6g7h8
    • App Secret（应用密钥）：点击"显示"才能看到，比如 xAbC123DeF456GhI789

立即复制保存这两个！

  • 打开记事本或备忘录
  • 分别粘贴App ID和App Secret
  • 特别注意：App Secret只能看一次，如果忘了就得重新生成

【配图7：标注App ID和App Secret的位置】

4.7 发布应用

凭证拿到后，发布应用：

  1. 应用管理页面，找到右上角的"创建版本"按钮
  2. 填写版本号（比如 1.0.0）和更新说明（随便写，比如"初始版本"）
  3. 点击"保存"，然后点击"申请发布"
  4. 提交后等待审核（企业自建应用通常几分钟就过，有的甚至秒过）

审核通过后，你的飞书应用就准备好了！

五、第三步：Clawdbot连接飞书（最后冲刺）

现在把Clawdbot和飞书连起来。

5.1 打开Clawdbot管理界面

还记得安装时终端输出的地址吗？（类似 http://localhost:3000）

  1. 浏览器打开那个地址
  2. 你会看到Clawdbot的Web管理界面
  3. 找到左侧菜单的"Channel管理"（或"渠道管理"）

【配图8：标注Clawdbot管理界面的布局，"Channel管理"位置】

5.2 安装飞书Channel插件

  1. 在Channel管理页面，找到"添加Channel"或"安装插件"
  2. 搜索框输入"飞书"或"Feishu"
  3. 找到飞书插件，点击"安装"

安装后会弹出配置界面。

5.3 配置飞书插件

这是最后的关键一步。

在配置界面，填入刚才从飞书拿到的凭证：

  • App ID：粘贴你保存的App ID
  • App Secret：粘贴你保存的App Secret
  • 其他选项：一般保持默认

填完后点击"保存配置"。

如果配置成功，Clawdbot会生成一个回调地址（Webhook URL），类似：

https://your-clawdbot-server.com/webhook/feishu

立即复制这个回调地址！

5.4 回到飞书填写回调地址

现在回到飞书开放平台（刚才的浏览器标签页）：

  1. 进入你的应用管理页面
  2. 左侧菜单找到"事件订阅"（还记得吗，我们之前留空的那个）
  3. 把Clawdbot生成的回调地址粘贴到"请求地址配置"输入框
  4. 点击"验证"按钮

如果一切正常，会显示"验证通过"（绿色提示）。

如果验证失败：

  • 检查Clawdbot服务是否在运行
  • 检查回调地址是否完整（没有多余空格）
  • 检查防火墙是否拦截

【配图9：Clawdbot的回调地址复制到飞书的验证流程】

✨ 回调地址就像门牌号，飞书通过它找到你的Clawdbot。

5.5 最终测试

一切配置完成！现在测试一下。

测试1：私聊测试

  1. 打开飞书
  2. 搜索你的机器人名称（比如"Clawdbot"）
  3. 打开私聊窗口，发送一条消息："你好"
  4. 等待几秒（首次可能稍慢）
  5. 机器人应该会回复你

测试2：群聊测试

  1. 创建一个测试群（或用现有群）
  2. 在群设置里，添加机器人（搜索你的机器人名称）
  3. 在群里@机器人，发送消息："@Clawdbot 介绍一下自己"
  4. 机器人应该会在群里回复

如果两个测试都通过，恭喜你，成功了！🎉

六、进阶使用与常见问题

6.1 如何切换AI模型？

Clawdbot的强大之处就在这里。

  1. 打开Clawdbot管理界面
  2. 找到"模型管理"或"LLM配置"
  3. 你可以添加多个模型（GLM、Minimax、GPT等）
  4. 切换默认模型，或者在对话时指定模型

每个模型有不同特点：

  • Qwen：快、免费、中文好
  • GLM：理解能力强，适合复杂对话
  • Minimax：便宜，适合高频调用

6.2 如何添加技能插件？

Clawdbot支持各种技能插件（Skill）：

  • 天气查询：问"北京天气怎么样"
  • 画图：让AI生成图片
  • 代码执行：直接运行代码片段
  • 网页搜索：实时搜索互联网

安装方法：

  1. Clawdbot管理界面 → "Skill Marketplace"
  2. 浏览或搜索你需要的技能
  3. 点击"安装"

6.3 常见问题排查

问题1：机器人不回复

可能原因：

  • ✅ 检查飞书权限是否全部开通（3个必需权限）
  • ✅ 检查事件订阅的回调地址是否正确
  • ✅ 查看Clawdbot日志是否有报错（终端里会显示）
  • ✅ 确认Clawdbot服务正在运行

问题2：机器人回复很慢

可能原因：

  • 尝试切换其他AI模型（Qwen通常最快）
  • 检查网络连接
  • 查看Clawdbot是否在处理大量请求

问题3：无法@机器人

可能原因：

  • 确认机器人已加入群聊（在群设置里添加）
  • 检查群聊权限im:message.group_at_msg是否开通

问题4：回调地址验证失败

可能原因：

  • 确认Clawdbot服务正在运行（浏览器能打开管理界面）
  • 检查回调地址是否完整复制（没有多余空格）
  • 检查服务器防火墙/端口配置
  • 尝试重新生成回调地址
