# 金句创作指南

## 什么是好的金句？

好的金句具备3个特征：
1. **独立性** - 脱离上下文也能理解
2. **传播性** - 让人想转发、想划线
3. **深度感** - 提供洞察，不是陈述事实

---

## 金句的6种类型

### 类型1: 对比/反转型

**结构：**
"人们以为X，其实是Y"
"不是A，而是B"

**示例：**
```
✅ "AI不会取代你。但会用AI的人，会。"
✅ "工具会过时，思维方式不会。"
✅ "技术的终点，是让技术消失。"
✅ "最难的不是学会使用AI，而是学会不依赖AI。"
```

**使用时机：**
- 揭示反常识的洞察
- 纠正常见误解
- 转折点

---

### 类型2: 比喻/类比型

**结构：**
"X就像Y"
"X是Y的Z"

**示例：**
```
✅ "提示词工程就像跟外星人聊天，你得猜它懂什么。"
✅ "训练AI模型是育儿，不是制造机器。"
✅ "代码是诗歌，只有诗人才懂另一个诗人的优雅。"
✅ "Debug是考古，你在挖掘三个月前的自己埋下的坑。"
```

**使用时机：**
- 解释抽象概念
- 让技术变得生动
- 开篇引入话题

---

### 类型3: 节奏/韵律型

**结构：**
使用重复、对仗、押韵

**示例：**
```
✅ "看得见的是代码，看不见的是思维。"
✅ "快的是机器，慢的是人。最快的，是人机协作。"
✅ "会写代码是技能，会不写代码是智慧。"
✅ "工具在手中，方法在心中。"
```

**使用时机：**
- 需要节奏感的段落
- 结尾总结
- 强调对立概念

---

### 类型4: 时间/演变型

**结构：**
"过去X，现在Y，未来Z"
"X的终点是Y"

**示例：**
```
✅ "十年前我们教电脑识字，现在电脑教我们写作。"
✅ "上一代人学打字，这一代人学提示词，下一代人可能什么都不用学。"
✅ "软件1.0是人写代码，2.0是人+AI写代码，3.0是AI写代码人审核。"
✅ "技术的终点，是让技术消失。"
```

**使用时机：**
- 讲历史发展
- 展望未来
- 说明趋势

---

### 类型5: 悖论/矛盾型

**结构：**
看似矛盾，实则深刻

**示例：**
```
✅ "最好的代码，是不用写的代码。"
✅ "最好的文档，是让人忘记它存在的文档。"
✅ "效率的最高境界，是不追求效率。"
✅ "学AI的第一步，是承认你不懂AI。"
```

**使用时机：**
- 表达深层洞察
- 引发思考
- 打破常规认知

---

### 类型6: 提问型

**结构：**
用问题引发思考

**示例：**
```
✅ "如果AI能做你的工作，那你的工作是什么？"
✅ "你在用工具，还是工具在用你？"
✅ "技术进步了，人进步了吗？"
✅ "当AI比你更懂你的代码，这是好事还是坏事？"
```

**使用时机：**
- 引入话题
- 引发深度思考
- 留有余味的结尾

---

## 金句创作的4个步骤

### 步骤1: 找到核心洞察

问自己：
- 这段想说的最重要的是什么？
- 有什么是反常识的？
- 有什么是读者可能没想到的？

### 步骤2: 选择合适的类型

根据内容选择金句类型：
- 有对比？→ 对比型
- 有抽象概念？→ 比喻型
- 有矛盾？→ 悖论型
- 有趋势？→ 时间型
- 想引发思考？→ 提问型

### 步骤3: 精简语言

删掉所有多余的词：
- 去掉"值得注意的是"、"我认为"等铺垫
- 用短句（一般不超过20字）
- 一句话只说一个意思

❌ 冗长版：
```
"我认为，在当前的技术发展趋势下，AI工具的发展速度
很快，但是我们的思维方式和方法论相对来说是更加稳定的，
不会那么容易过时。"
```

✅ 精简版：
```
"工具会过时，思维方式不会。"
```

### 步骤4: 打磨节奏

调整用词，让句子有节奏感：
- 对称结构（X的X，Y的Y）
- 重复关键词
- 适当的停顿（用逗号和句号）

## 金句的位置和密度

### 最佳位置（优先级递减）：

1. **开篇第一段** - 抓住注意力
   ```
   示例：
   "AI不会取代你。但会用AI的人，会。

   这不是危言耸听，而是正在发生的事实。[展开...]"
   ```

2. **核心洞察之后** - 强化记忆
   ```
   示例：
   "[解释一个复杂概念...]

   说白了：工具在手中,方法在心中。"
   ```

3. **段落/小节结尾** - 画龙点睛
   ```
   示例：
   "[讨论技术演变...]

   技术的终点，是让技术消失。"
   ```

4. **文章结尾** - 留有余味
   ```
   示例：
   "[总结全文...]

   如果AI能做你的工作，那你的工作是什么？
   这是每个人都该思考的问题。"
   ```

### 密度控制：

根据字数档位：

| 档位 | 字数 | 金句数 | 密度（字/句） |
|------|------|--------|---------------|
| A. 精炼版 | 1200-2000 | 2-3句 | 600-800 |
| B. 标准版 | 2000-4000 | 3-5句 | 500-800 |
| C. 深度版 | 4000-7000 | 5-7句 | 600-1000 |
| D. 长文版 | 7000-10000 | 6-8句 | 900-1400 |

**原则：**
- 不要连续两个段落都有金句
- 金句之间至少间隔2-3个段落
- 宁可少而精，不要多而滥

## 常见错误

### 错误1: 为了金句而金句

❌ 问题：
```
每个段落都试图升华，导致：
- 失去重点
- 读者疲劳
- 显得刻意
```

✅ 解决：
只在真正有洞察的地方用金句。
有些事实就该平铺直叙。

### 错误2: 太长

❌ 问题：
```
"在AI快速发展的今天，我们需要思考的不仅仅是
如何使用这些工具，更重要的是如何在工具的辅助下
保持自己的独立思考能力和创造力。"
（43字，太长了）
```

✅ 解决：
精简到20字以内：
```
"AI是工具，思考是本能。"（9字）
```

### 错误3: 太抽象

❌ 问题：
```
"技术赋能，创新驱动，智慧引领。"
（听起来很高大上，但不知道在说什么）
```

✅ 解决：
用具体的意象：
```
"代码是砖，架构是图纸，最终要盖的是房子。"
```

### 错误4: 陈述事实

❌ 问题：
```
"AI正在改变世界。"
（这是事实，不是洞察）
```

✅ 解决：
提供新视角：
```
"AI不是在改变世界，是在改变我们理解世界的方式。"
```

## 金句训练方法

### 练习1: 改写事实为金句

把平淡的陈述改写为金句：

陈述：AI可以帮助程序员提高效率。
金句：AI是助手，不是替身。

陈述：学习AI需要时间。
金句：快的是工具，慢的是理解。

### 练习2: 一句话总结

读一段技术文章，用一句话（<20字）总结核心思想。

### 练习3: 日常观察

看到任何技术现象，问自己：
- 这个现象的本质是什么？
- 有什么反常识的地方？
- 用一句话怎么说？

---

## 金句检查清单

完成金句后，检查：

- [ ] 是否能脱离上下文独立理解？
- [ ] 是否提供了新的洞察（不只是陈述事实）？
- [ ] 是否简短有力（<20字）？
- [ ] 是否有节奏感？
- [ ] 是否让你想转发或划线？
- [ ] 是否显得刻意或矫情？（如果是，删掉）
- [ ] 整篇文章的金句密度是否合理？

---

## 记住

**金句的作用：**
在恰当的时刻，用简洁有力的语言，
点亮读者的思维，留下深刻印象。

**不是：**
每段都要升华，每句都要深刻。

**标准：**
读者读到这句话时，会停下来想一想，
也许还会划线，甚至发朋友圈。
