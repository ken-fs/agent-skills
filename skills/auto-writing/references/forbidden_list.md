# AI写作禁止清单

## 为什么需要这个清单？

AI生成的文章往往有明显的"机器味"，表现为：
- 过度程式化的结构
- 重复使用固定套路
- 缺乏个性和真实感
- 让读者一眼就看出是AI写的

这个清单帮助你识别和避免这些模式，让文章更像人写的。

---

## 🚫 禁止的开头模式

### 禁止1: "在当今社会..." 系列

❌ **禁止：**
```
"在当今社会，人工智能技术日新月异..."
"在数字化时代，AI已经成为..."
"随着科技的发展，越来越多的人开始..."
```

✅ **替代方案：**
```
"去年我第一次用ChatGPT时，它给我写了一首藏头诗。
虽然有点蹩脚，但那一刻我意识到，AI真的不一样了。"

"上周三，李明的工作被一个AI抢了。不是科幻小说，
是真实发生的事。"

"你打开手机，问Siri今天天气。这个简单的动作背后，
有一整套AI系统在运转。"
```

### 禁止2: "众所周知..." 系列

❌ **禁止：**
```
"众所周知，AI技术..."
"大家都知道，机器学习..."
"显而易见，深度学习..."
```

✅ **替代方案：**
```
"大多数人以为AI就是聊天机器人。其实，这只是冰山一角。"

"如果你认为AI只是个工具，那你可能低估了它的影响。"
```

### 禁止3: "让我们来看看..." 系列

❌ **禁止：**
```
"让我们来看看AI是如何工作的..."
"接下来让我们了解一下..."
"现在让我们深入探讨..."
```

✅ **替代方案：**
```
直接进入内容，不需要过渡。

"AI的工作原理其实很简单：喂数据，找规律，做预测。"
```

---

## 🚫 禁止的结构模式

### 禁止4: "首先...其次...最后..." 三段论

❌ **禁止：**
```
"首先，我们需要了解AI的基本概念。
其次，我们要掌握使用方法。
最后，我们来看实际应用。"
```

✅ **替代方案：**
```
用自然的过渡：
"理解AI，先从它的工作原理说起。"
"掌握了原理，再看怎么用。"
"知道怎么用了，就能解决实际问题。"

或者不需要过渡，直接进入：
"AI的工作原理：数据 → 模式 → 预测。"
```

### 禁止5: 机械的小标题序列

❌ **禁止：**
```
## 一、背景介绍
## 二、核心概念
## 三、实践应用
## 四、未来展望
```

✅ **替代方案：**
```
用有吸引力的小标题：
## 为什么程序员开始担心失业？
## AI不是魔法，是数学
## 三个案例，看懂AI能做什么
## 下一步会发生什么？
```

### 禁止6: 过度使用排比

❌ **禁止：**
```
"AI改变了我们的工作方式，
改变了我们的生活方式，
改变了我们的思维方式。"
```

✅ **替代方案：**
```
偶尔使用，制造节奏感，但不要每段都用。

"AI改变了工作方式。也改变了我们对'工作'的定义。"
```

---

## 🚫 禁止的过渡表达

### 禁止7: "值得注意的是..." 系列

❌ **禁止：**
```
"值得注意的是..."
"需要指出的是..."
"值得一提的是..."
"这里需要强调的是..."
```

✅ **替代方案：**
```
"有个有意思的地方..."
"你可能没注意到..."
"这里有个细节..."
"说个你不知道的..."
```

### 禁止8: "总而言之..." 系列

❌ **禁止：**
```
"总而言之..."
"综上所述..."
"总的来说..."
"概括而言..."
```

✅ **替代方案：**
```
"说到底..."
"简单说..."
"核心就一句话..."

或者直接说结论，不需要过渡词。
```

---

## 🚫 禁止的修辞模式

### 禁止9: 过度的感叹号

❌ **禁止：**
```
"这真是太神奇了！AI居然可以做到这些！
简直不可思议！未来充满了无限可能！"
```

✅ **替代方案：**
```
一篇文章最多3-5个感叹号。

"这真的很神奇。AI做到了我们以为不可能的事。"
```

### 禁止10: 空洞的形容词堆砌

❌ **禁止：**
```
"革命性的、突破性的、颠覆性的、创新性的技术..."
"强大的、优秀的、卓越的、出色的性能..."
```

✅ **替代方案：**
```
用具体的数据和例子：

"GPT-4的参数量是GPT-3的10倍。这意味着什么？
它能记住的'知识'多了10倍。"

"Claude Code能在30秒内读完1万行代码，
找出你可能花3小时才能发现的bug。"
```

### 禁止11: "简单来说..." 然后说得一点也不简单

❌ **禁止：**
```
"简单来说，Transformer架构通过自注意力机制实现了
并行化的序列建模，利用位置编码保留序列信息..."
```

✅ **替代方案：**
```
真正简化：

"Transformer就像一个超级记事本，能同时看到一句话
里的所有词，找出它们之间的关系。"

如果无法简化，就别说"简单来说"。
```

---

## 🚫 禁止的结尾模式

### 禁止12: 空洞的号召

❌ **禁止：**
```
"让我们一起拥抱AI时代！"
"未来已来，你准备好了吗？"
"让科技改变生活，让AI造福人类！"
```

✅ **替代方案：**
```
给出具体的行动建议：

"如果你想开始学AI，从这三件事做起：
1. 注册一个ChatGPT账号，每天用它解决一个问题
2. 看一门Andrew Ng的机器学习课程
3. 找一个小项目，试着用AI解决它"

或者留有余味的结尾：

"AI会取代程序员吗？可能会。
但更可能的是，AI会创造出我们现在想象不到的新职业。
就像100年前，没人知道会有'产品经理'这个岗位。"
```

### 禁止13: 重复摘要

❌ **禁止：**
```
"本文介绍了AI的基本概念、工作原理和应用场景。
通过学习本文，读者可以了解到AI的强大功能，
并掌握如何使用AI来提升工作效率..."
```

✅ **替代方案：**
```
不要重复已经说过的内容，给新的洞察：

"AI不会取代你。但会用AI的人，会。"

"技术的细节会过时，但今天聊的这个思维方式，
十年后依然有用。"
```

## 🚫 禁止的语言习惯

### 禁止14: 被动语态泛滥

❌ **禁止：**
```
"AI技术被广泛应用于..."
"该模型被训练于..."
"这个问题可以被解决..."
```

✅ **替代方案：**
```
主动语态更有力：

"各行各业都在用AI。"
"研究者用几百万篇文章训练这个模型。"
"你可以用AI解决这个问题。"
```

### 禁止15: 过度的委婉和不确定

❌ **禁止：**
```
"可能..."、"也许..."、"似乎..."、"有可能..."、
"在某种程度上..."、"一定程度上..."
```

✅ **替代方案：**
```
确定的地方要确定：
"AI会改变软件开发。" （不是"可能会"）

不确定的地方明说：
"没人知道AGI什么时候出现。2030年？2050年？
谁都说不准。"
```

## ✅ 检测方法

写完文章后，用这些方法检测：

### 方法1: 关键词搜索
搜索这些词，看出现次数：
- "首先"、"其次"、"最后"
- "值得注意"、"需要指出"
- "总而言之"、"综上所述"
- "让我们"
- "众所周知"、"显而易见"

如果每篇文章出现超过2次，需要改写。

### 方法2: 朗读测试
大声读出来。如果听起来像：
- 新闻播报
- 说明书
- 学术论文

那就需要改得更口语化。

### 方法3: 朋友测试
给朋友看一段，问：
"这像人写的，还是AI写的？"

如果回答是AI，找到原因并修改。

### 方法4: 开头测试
只看文章前100字。
- 如果是背景介绍，❌ 删掉重写
- 如果是场景/故事/问题，✅ 通过

### 方法5: 金句密度测试
数一数有多少句子：
- 可以独立理解
- 让人想划线
- 可以发朋友圈

如果超过字数档位规定，减少金句。
如果太少，在关键位置增加。

---

## 记住

**核心原则：**
写得像个真实的人，在和朋友聊天，分享自己的理解和见解。

**不是：**
一个AI，在机械地整理和输出信息。

**测试标准：**
你的朋友读完后，会说"写得不错"，而不是"这是AI写的吧"。
