# 七维度文章审核标准

## 审核流程

完成文章初稿后，从7个维度进行全面审核，每个维度1-10分评分。

**目标：**
- 所有维度≥7分才能发布
- <7分的维度需要针对性优化

---

## 维度1: 可读性 (Readability)

### 评分标准

**9-10分：**
- 句子简洁，平均15-20字
- 段落结构清晰，3-5句/段
- 过渡自然，阅读流畅
- 无需反复阅读就能理解

**7-8分：**
- 句子长度适中，偶有过长
- 段落结构基本合理
- 大部分过渡自然
- 个别地方需要重读

**5-6分：**
- 句子普遍偏长（>30字）
- 段落过长或过短
- 过渡生硬
- 多处需要重读理解

**<5分：**
- 句子冗长难懂
- 段落结构混乱
- 逻辑跳跃
- 阅读困难

### 检查项

- [ ] 句子长度（平均<25字）
- [ ] 段落结构（3-5句/段）
- [ ] 过渡词使用（自然、不生硬）
- [ ] 标点符号（逗号不过度使用）
- [ ] 专业术语（首次出现有解释）

### 常见问题及修复

**问题1: 句子过长**
```
❌ 原句（45字）：
"AI模型通过学习大量的训练数据来识别其中的模式和规律，
然后在面对新的输入时利用这些学到的知识进行预测或生成，
这个过程类似于人类的学习方式但速度要快得多。"

✅ 修复（拆分为3句）：
"AI模型怎么工作？它先学习大量训练数据，找出其中的规律。
遇到新问题时，就用学到的知识做预测。这个过程很像人类学习，
只是快了几百倍。"
```

**问题2: 段落过长**
```
❌ 原文（一个段落8句话，200字）

✅ 修复：拆分为2-3个段落
- 第一段：引入问题（2-3句）
- 第二段：展开说明（3-4句）
- 第三段：总结/过渡（1-2句）
```

## 维度2: 技术准确性 (Technical Accuracy)

### 评分标准

**9-10分：**
- 所有技术概念准确无误
- 术语使用规范
- 数据/引用有来源
- 案例真实可验证

**7-8分：**
- 核心概念准确
- 个别术语不够精确
- 大部分信息可靠
- 案例基本真实

**5-6分：**
- 有明显技术错误
- 术语使用混乱
- 信息来源不明
- 案例可能虚构

**<5分：**
- 严重技术错误
- 概念理解错误
- 信息不可靠
- 误导读者

### 检查项

- [ ] 技术概念定义正确
- [ ] 术语使用规范（不自创术语）
- [ ] 数据有来源（标注出处）
- [ ] 案例真实（可验证）
- [ ] 时间信息准确（版本号、发布时间）

### 验证方法

1. **概念验证：**
   - 对照官方文档
   - 多个权威来源交叉验证

2. **数据验证：**
   - 检查数据来源
   - 验证时效性（<1年）
   - 标注引用

3. **案例验证：**
   - 确认案例真实性
   - 保留原始链接

### 常见错误

❌ **错误1: 概念混淆**
```
"GPT是一种深度学习算法"
（GPT是模型，不是算法）

正确：
"GPT是基于Transformer架构的语言模型"
```

❌ **错误2: 数据过时**
```
"ChatGPT的训练数据截至2021年9月"
（这是GPT-3.5的信息，已过时）

正确：
"GPT-4的训练数据截至2023年12月"（需验证最新版本）
```

## 维度3: 逻辑连贯性 (Logical Coherence)

### 评分标准

**9-10分：**
- 论证链条完整
- 因果关系清晰
- 结构严密
- 无逻辑跳跃

**7-8分：**
- 整体逻辑通顺
- 个别跳跃但可理解
- 因果关系基本清楚

**5-6分：**
- 有明显逻辑漏洞
- 因果关系不清
- 结构松散

**<5分：**
- 逻辑混乱
- 前后矛盾
- 无法理解

### 检查项

- [ ] 大纲结构合理（总-分-总）
- [ ] 每个Section服务于主题
- [ ] 段落间有逻辑关系
- [ ] 结论有论据支持
- [ ] 无前后矛盾

### 逻辑检查方法

1. **画逻辑图：**
```
主题
 ├─ 论点1
 │   ├─ 论据A
 │   └─ 论据B
 ├─ 论点2
 │   ├─ 论据C
 │   └─ 论据D
 └─ 结论
```

2. **检查因果链：**
- 每个"所以"前面有"因为"
- 每个结论有论据支持

3. **反向阅读：**
- 从结论往回推
- 看论证是否成立

## 维度4: 反AI味 (Human-like Writing)

### 评分标准

**9-10分：**
- 完全像人写的
- 有个性和温度
- 无AI写作套路
- 自然真实

**7-8分：**
- 基本像人写的
- 偶有AI味
- 大部分表达自然

**5-6分：**
- 明显AI味
- 套路表达多
- 缺乏个性

**<5分：**
- 典型AI生成
- 机械化
- 无温度

### 检查项（禁止清单）

- [ ] "首先、其次、最后"（出现<2次）
- [ ] "值得注意的是"、"需要指出的是"（0次）
- [ ] "总而言之"、"综上所述"（0次）
- [ ] "让我们来看看"（0次）
- [ ] "众所周知"、"显而易见"（0次）
- [ ] 过度排比（连续3段以上，0次）
- [ ] 感叹号过多（全文<5个）

### 人性化技巧

✅ **使用第一/第二人称：**
```
"我第一次用Claude Code时..."
"你可能遇到过这种情况..."
"我们来看个例子..."
```

✅ **分享个人经验：**
```
"说实话，这个功能我也经常用错。"
"有次我debug了3小时，最后发现..."
```

✅ **用口语化表达：**
```
"有个有意思的地方..."
"说白了..."
"换句话说..."
"其实..."
```

## 维度5: 风格一致性 (Style Consistency)

### 评分标准

**9-10分：**
- 完美符合选定风格
- 体现"卡夫卡+余华+马尔克斯"
- 风格统一

**7-8分：**
- 基本符合风格
- 偶有偏离
- 整体和谐

**5-6分：**
- 风格不统一
- 多处偏离
- 混杂其他风格

**<5分：**
- 完全不符合风格
- 风格混乱

### 风格检查清单

**卡夫卡的精准：**
- [ ] 技术术语准确
- [ ] 逻辑推导严密
- [ ] 每个形容词必要

**余华的朴素：**
- [ ] 短句为主（平均<20字）
- [ ] 用词简单直接
- [ ] 无华丽修辞

**马尔克斯的神奇：**
- [ ] 有意外的比喻
- [ ] 技术变得有故事性
- [ ] 平凡中见神奇

### 风格示例

✅ **好的风格融合：**
```
"训练AI模型，就像养一个天才孩子。（马尔克斯）

你喂给它整个互联网的知识。它学什么，你控制。（余华）
但它学到什么、会说什么，你永远无法完全预测。（余华）

这不是bug，这是特性。（卡夫卡）"
```

## 维度6: 受众匹配度 (Audience Fit)

### 评分标准

**9-10分：**
- 完美匹配目标受众
- 难度适配
- 案例贴合
- 语言亲和

**7-8分：**
- 基本匹配受众
- 难度稍有偏差
- 大部分内容贴合

**5-6分：**
- 受众不清晰
- 难度不合适
- 案例不贴合

**<5分：**
- 完全不匹配
- 难度错位
- 脱离受众

### 受众：普通大众、程序员、AI爱好者

**检查项：**

- [ ] 技术术语有解释（考虑普通大众）
- [ ] 有代码示例（满足程序员）
- [ ] 有前沿趋势（满足AI爱好者）
- [ ] 难度递进（从浅到深）
- [ ] 案例多样（覆盖不同背景）

### 难度把控

**三层结构：**

1. **浅层（普通大众能懂）：**
```
"AI就像一个超级勤奋的学生，看了几百万篇文章，
记住了哪些词经常一起出现。"
```

2. **中层（程序员感兴趣）：**
```
"技术上，这叫统计语言模型。通过计算词频和共现概率，
建立概率分布。"
```

3. **深层（AI爱好者想深入）：**
```
"具体实现用的是Transformer架构的多头注意力机制，
能并行处理序列信息。感兴趣可以看论文《Attention Is All You Need》。"
```

## 维度7: 吸引力 (Engagement)

### 评分标准

**9-10分：**
- 标题吸引人
- 开篇抓眼球
- 金句恰到好处
- 结尾有余味
- 想读完并分享

**7-8分：**
- 标题中规中矩
- 开篇尚可
- 金句合理
- 结尾平淡

**5-6分：**
- 标题平淡
- 开篇无聊
- 金句生硬
- 结尾草率

**<5分：**
- 标题无吸引力
- 开篇劝退
- 无金句
- 虎头蛇尾

### 检查项

**标题检查：**
- [ ] 有悬念/问题/冲突
- [ ] 简洁（<20字）
- [ ] 不标题党
- [ ] 包含关键词

**开篇检查：**
- [ ] 前100字抓住注意力
- [ ] 场景/故事/问题开头
- [ ] 避免背景介绍

**金句检查：**
- [ ] 数量符合档位
- [ ] 位置恰当
- [ ] 质量高（想划线）
- [ ] 不刻意

**结尾检查：**
- [ ] 回归开篇
- [ ] 有行动建议
- [ ] 留有余味
- [ ] 不重复摘要

---

## 综合评分

### 计算方法

总分 = (7个维度分数之和) / 7

### 发布标准

- **优秀**：平均分≥9分，所有维度≥8分
- **良好**：平均分≥8分，所有维度≥7分
- **合格**：平均分≥7分，所有维度≥6分
- **需优化**：任意维度<6分

### 优化策略

**针对性优化：**
1. 找到<7分的维度
2. 重点优化该维度
3. 再次评分
4. 直到所有维度≥7分

**优先级：**
1. 技术准确性（最重要，必须≥9分）
2. 可读性（影响阅读体验）
3. 反AI味（影响真实感）
4. 其他维度

```markdown
| 6. 受众匹配度 | 9/10 | ✅ 通过 |

| **平均分** | **7.7/10** | ✅ 合格 |

## 详细分析

### 维度4: 反AI味 (6/10) - 需优化

**发现问题：**
1. 第2段使用"首先...其次...最后..."结构
2. 第5段出现"值得注意的是"
3. 第8段排比过度

**优化建议：**
1. 改写第2段，用自然过渡代替"首先其次"
2. 删除"值得注意的是"，直接说内容
3. 简化第8段排比，保留1-2个即可

### 其他亮点
- 技术准确性高，所有概念验证无误
- 案例贴合受众，覆盖不同背景
- 金句恰到好处，位置合理

## 行动建议
1. 优先修复反AI味问题（预计15分钟）
2. 修复后重新评分
3. 确保所有维度≥7分后发布
```

## 记住

**审核的目的：**
确保文章达到"大师级"标准，值得读者的时间。

**审核的态度：**
严格但不苛刻，追求卓越但接受"足够好"。

**审核的标准：**
所有维度≥7分 = 可以发布
平均分≥9分 = 大师级作品
